## Design Context

### Users
Building managers and compliance consultants working on Washington, DC Building Energy Performance Standards (BEPS) compliance. They operate under regulatory deadlines, switching between checking compliance status, running verification workflows, and preparing filing packets. They need the interface to feel reliable and immediately legible — data density matters, but clarity comes first.

### Brand Personality
**Precise · Trustworthy · Modern**

The interface should evoke **trust and clarity** above all else. Users are making consequential decisions about regulatory compliance; the UI must feel authoritative without being intimidating, and clean without being sterile.

### Aesthetic Direction
- **Visual tone**: Clean, professional, premium — a polished compliance tool, not a consumer dashboard
- **Color palette**: Slate/neutral base with cyan-to-emerald gradient accents. HSL CSS variable system via shadcn/ui (radix-nova style, neutral base color)
- **Typography**: Inter (sans-serif, body/UI) + JetBrains Mono (monospace, data values and badges). Tight tracking, tabular-nums for data alignment
- **Components**: Machined glass cards with subtle multi-layer shadows, recessed inputs with inset shadows, luminous gradient buttons (cyan → emerald), eco-tech pill badges in monospace, gradient accent strips for section hierarchy
- **Theme**: Light mode only
- **Motion**: Subtle, purposeful — Framer Motion for page/component transitions, tailwindcss-animate for micro-interactions. Nothing playful or attention-grabbing
- **Data visualization**: Recharts for charts, Mapbox GL for geospatial views
- **Icons**: Lucide React
- **Border radius**: 0.5rem base (`--radius`), with `rounded-2xl` for cards and `rounded-full` for pills/buttons
- **Anti-references**: None explicitly stated, but the design language is firmly professional-technical — avoid gamification, playful illustrations, or consumer-app aesthetics

### Design Principles
1. **Clarity over decoration** — Every element must earn its place. Prioritize readable data, clear hierarchy, and predictable patterns over ornamental styling.
2. **Trust through precision** — Monospace data values, tabular alignment, versioned compliance runs, and audit trails reinforce that the system is exact and reliable.
3. **Quiet sophistication** — The eco-tech palette (cyan/emerald) is used as a controlled accent, not a dominant theme. The machined glass aesthetic signals quality without shouting.
4. **Data density with breathing room** — Building managers need to scan many values quickly. Use generous whitespace and consistent spacing to make dense information feel calm rather than overwhelming.
5. **Progressive disclosure** — Surface status and summaries first; let users drill into detail through clean workflows rather than cramming everything onto one screen.
